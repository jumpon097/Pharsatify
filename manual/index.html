3. ตั้งค่า GAS

ค่าที่ต้องเปลี่ยนใน Code.gs

const APP_CONFIG = Object.freeze({
  FORM_ID: 'FORM_ID_ใหม่',
  SPREADSHEET_ID: 'SPREADSHEET_ID_ใหม่',
  RESPONSE_SHEET_ID: 123456789,

  APP_TITLE: 'ประเมินความพึงพอใจ',
  SERVICE_NAME: 'ชื่อห้องหรือหน่วยงาน',
  HOSPITAL_NAME: 'โรงพยาบาลมะเร็งอุบลราชธานี',

  DEFAULT_CHAT_ID: 'CHAT_ID_ใหม่'
});
สำคัญ: RESPONSE_SHEET_ID คือเลข gid ของแท็บตอบกลับ ไม่ใช่ Spreadsheet ID

Telegram

เก็บ Bot Token ใน Script Properties เท่านั้น ห้ามใส่ Token ใน GitHub

TELEGRAM_BOT_TOKEN = Token ของ Bot
TELEGRAM_CHAT_ID = Chat ID ของกลุ่ม

รัน setupTelegramConfig() และ authorizeSystem() อย่างละหนึ่งครั้ง

Deploy

  1. Deploy → New deployment
  2. Type: Web app
  3. Execute as: Me
  4. Who has access: Anyone
  5. Deploy และคัดลอก URL ที่ลงท้าย /exec

เมื่อแก้ GAS ให้ใช้ Deploy → Manage deployments → Edit → New version → Deploy เพื่อรักษา URL เดิม

4. สร้าง PWA subfolder

คัดลอกไฟล์ต่อไปนี้เข้า subfolder ใหม่

index.html
manifest.webmanifest
sw.js

เปลี่ยน GAS URL

<iframe id="appFrame" src="GAS_URL_ใหม่"></iframe>

เปลี่ยนชื่อใน manifest

{
  "name": "ประเมินความพึงพอใจ ชื่อหน่วยงาน",
  "short_name": "ประเมินชื่อย่อ",
  "start_url": "./",
  "scope": "./",
  "display": "standalone"
}
แต่ละ subfolder ต้องมี manifest และ service worker ของตนเอง เพื่อให้ติดตั้งเป็น PWA คนละแอป

5. การแบ่งหน้าที่

ผู้ดูแลกลางเก็บ Code.gs ต้นแบบ ดูแล GitHub และ Deploy
ผู้รับผิดชอบหน่วยงานสร้าง Form/Sheet และแจ้งชื่อหน่วยงานกับ Chat ID
ผู้ทดสอบตรวจชีต Telegram และการกลับหน้าประเมินใน 5 วินาที

6. Checklist ก่อนเปิดใช้

  • คะแนนบันทึกลงชีตถูกแท็บ
  • ข้อเสนอแนะบันทึกถูกคอลัมน์
  • Telegram ส่งเข้ากลุ่มถูกต้อง
  • หน้า PWA แสดงชื่อหน่วยงานถูกต้อง
  • หน้าขอบคุณกลับหน้าประเมินใน 5 วินาที
  • ทดลองติดตั้ง PWA บนมือถือหรือคอมพิวเตอร์
  • ไม่มี Bot Token อยู่ใน GitHub

แก้ไขภายหลัง

  • แก้หน้าตา: แก้ Index.html ใน GAS แล้ว Deploy New version
  • เปลี่ยน Form/Sheet/Chat ID: แก้ Code.gs แล้ว Deploy New version
  • เปลี่ยน GAS URL: แก้ index.html ใน subfolder
  • PWA ยังแสดงของเก่า: เพิ่มเลขเวอร์ชันใน CACHE_NAME ของ sw.js
